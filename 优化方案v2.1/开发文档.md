## 1. 系统概述

### 1.1 项目背景

智慧校园微服务系统是一个基于 Spring Cloud 微服务架构的综合性校园管理平台。系统涵盖教务管理（课程、排课、选课、成绩、补考、重修、考试、考勤、评价）、人员管理（学生、教师、管理员统一认证）、图书管理（图书分类、借阅、阅读报告）、座位预约、请假审批（学生请假 + 教师调课）、通知公告、社团活动、奖学金管理等核心业务。

v3.0 从单体架构重构为微服务架构，各业务域独立部署、独立扩展、独立迭代。

### 1.2 用户角色定义

| 角色类型 | 说明 | 访问端 |
|---------|------|--------|
| 超级管理员（SuperAdmin） | 拥有全部管理权限，可管理其他管理员 | 管理端 |
| 管理员（Admin） | 系统管理员，拥有管理权限 | 管理端 |
| 教师（Teacher） | 授课教师，可审批请假、录入成绩、发起考勤、申请调课 | 管理端 + 用户端 |
| 学生（Student） | 在校学生，使用系统各项服务 | 用户端 |

### 1.3 微服务技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 服务框架 | Spring Boot 3 + Spring Cloud 2024.x | 微服务基础框架 |
| 服务注册与发现 | Nacos | 替代 Eureka，兼顾配置中心 |
| API 网关 | Spring Cloud Gateway | 统一入口、路由、鉴权、限流 |
| 远程调用 | OpenFeign + Sentinel | 声明式 HTTP 调用 + 熔断降级 |
| 数据库 | MySQL 8.0 | 每服务独立 database（当前阶段共享实例） |
| ORM | MyBatis-Plus 3.5.x | 增强 MyBatis，简化 CRUD |
| 缓存 | Redis + Caffeine | 分布式缓存 + 本地缓存二级缓存 |
| 消息队列 | RabbitMQ | 异步解耦、事件驱动 |
| 认证鉴权 | JWT（jjwt 0.12.x） | 网关统一验签，服务间 Feign 传递 |
| 密码加密 | BCrypt | 不可逆加密 |
| 分页 | MyBatis-Plus 分页插件 | 替代 PageHelper |
| 文档 | SpringDoc OpenAPI 3 | 自动生成 Swagger 文档 |
| 对象存储 | 阿里云 OSS / MinIO | 文件上传存储 |
| Excel | Apache POI / EasyExcel | 数据导入导出 |
| 实时推送 | WebSocket + STOMP | 通知、考勤等实时推送 |
| 可观测性 | Micrometer + Prometheus + Grafana | 服务监控与告警 |

### 2.5 WebSocket 实时推送

WebSocket 是系统级实时通信基础设施，通过 STOMP 协议实现服务端向客户端主动推送消息。在微服务架构中，WebSocket 连接管理可作为 `gateway-service` 的内置能力，或独立为 `websocket-service`。

#### 2.5.1 推送场景

| 场景 | 触发事件 | 推送目标 | 消息内容 |
|------|---------|---------|---------|
| 通知推送 | 管理员发布通知 | 全体/指定范围用户 | 通知标题 + 摘要 + 链接 |
| 考勤提醒 | 教师发起考勤 | 该课程所有学生 | 课程名称 + 签到截止时间 |
| 审批结果 | 请假/调课审批完成 | 申请人 | 审批结果 + 审批意见 |
| 调课通知 | 教师调课审批通过 | 该课程所有学生 | 原时间 + 新时间 + 新教室 |
| 选课提醒 | 选课时段开启 | 可选课学生 | 选课开始/结束时间 |
| 借阅催还 | 图书逾期未还 | 借阅学生 | 书名 + 逾期天数 |

#### 2.5.2 技术实现

| 组件 | 说明 |
|------|------|
| 协议 | WebSocket + STOMP（Simple Text Oriented Messaging Protocol） |
| 消息中间件 | RabbitMQ（各服务发布事件 → WebSocket 处理器消费 → 推送到对应客户端） |
| 连接管理 | 用户登录后建立 WebSocket 连接，Redis 维护 user → channel 映射 |
| 心跳机制 | 每30秒发送心跳帧，超时自动断开并重连 |
| 离线补偿 | 用户离线期间消息存入 Redis List，重连后补推 |

#### 2.5.3 STOMP 推送通道

| 通道 | 订阅者 | 说明 |
|------|--------|------|
| `/user/queue/notification` | 所有用户 | 个人通知消息 |
| `/topic/attendance/{scheduleId}` | 选课学生 | 考勤签到通知 |
| `/user/queue/approval` | 请假申请人 | 请假/调课审批结果 |
| `/topic/course/{scheduleId}` | 选课学生 | 调课/停课通知 |

#### 2.5.4 事件驱动流程

```
teaching-service ──(考勤发起)──→ RabbitMQ ──→ WebSocket Handler ──→ 学生客户端
leave-service ────(审批结果)──→ RabbitMQ ──→ WebSocket Handler ──→ 申请人客户端
system-service ───(通知发布)──→ RabbitMQ ──→ WebSocket Handler ──→ 目标用户客户端
library-service ──(借阅催还)──→ RabbitMQ ──→ WebSocket Handler ──→ 学生客户端
```

**核心变更（v3.0）**:

- 教师请假时需指定需要调整的排课记录 (`course_schedule_id`)，并填写调课后的时间/教室

| endTime | LocalDateTime | @NotNull | 结束时间 |
| reason | String | @NotBlank | 请假/调课事由 |

**教师调课专属字段**（applicantType=TEACHER 时填写）:

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| courseScheduleId | Long | @NotNull | 需要调整的排课ID |

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| leaveRequestId | Long | @NotNull | 请假申请ID |
| approverId | Long | @NotNull | 审批人ID |
| result | String | @NotBlank | 审批结果：APPROVED / REJECTED |
| comment | String | — | 审批意见 |

#### 3.8.2 VO 定义

**LeaveRequestDetailVO** — LeaveRequestListVO全部字段 + 调课信息(newDate/newWeekDay/newStartSection/newEndSection/newClassroomId), approvalHistory(List<LeaveApprovalLogVO>)

#### 3.8.3 业务规则

1. 学生请假流程：学生提交 → 教师审批 → 完成
2. 教师调课流程：教师提交 → 管理员审批 → 通过后自动更新排课
3. 教师调课需校验新时间/教室是否冲突
4. 审批通过后，通过 RabbitMQ 发送事件通知 `teaching-service` 更新排课
5. 请假开始时间必须早于结束时间
6. 同一排课在同一时间只能有一条调课申请

## 6. 业务规则

### 6.1 选课规则

1. 学生只能在管理员设定的选课时段内选课/退课
2. 选修/公选课程有容量限制，先到先得
3. 同一课程同一学期不可重复选课（唯一约束兜底）
4. 必修课程由管理员批量导入，学生无需自选

### 6.2 成绩规则

1. 总评成绩 = 平时成绩 × 30% + 期末成绩 × 70%（可配置）
2. 总评成绩 < 60 或 exam_status 为"缺考" → 进入补考名单
3. 补考成绩替代期末成绩重新计算总评，上限60分
4. 补考未通过 → 进入重修名单

### 6.3 排课规则

1. 同一教室同一时段不可排两门课（唯一约束兜底）
2. 同一教师同一时段不可排两门课
3. 教师调课审批通过后自动更新排课，需校验新时段是否冲突

### 6.4 借阅规则

1. 最大同时借阅册数：5本（可配置）
2. 最大借阅天数：30天（可配置）
3. 逾期未还 → 禁止新借阅，直到归还

### 6.5 座位预约规则

1. 每天22:00后可预约次日座位
2. 预约后需在开始时间前后30分钟内签到，否则释放
3. 暂离不超过30分钟

### 6.6 请假/调课规则

1. 学生请假需请假时间 ≤ 当前学期结束日期
2. 教师调课需在排课时间前至少24小时提交
3. 同一排课同一时段仅允许一条待审批调课申请

## 7. 异常处理规范

### 7.1 微服务异常处理

| 场景 | 处理方式 |
|------|---------|
| 服务调用超时 | Feign + Sentinel 熔断降级，返回降级响应 |
| 服务不可用 | Sentinel 快速失败，返回 503 |
| 数据库连接失败 | 全局异常拦截 → 500 + 告警 |
| 参数校验失败 | MethodArgumentNotValidException → 400 + 详细错误信息 |
| 认证失败 | Gateway 全局拦截 → 401 |
| 权限不足 | AOP 切面拦截 → 403 |
| 业务异常 | BusinessException → 对应错误码 + 提示信息 |

### 7.2 异常日志

- 所有异常记录 ERROR 级别日志
- 业务异常记录 WARN 级别日志
- 日志中包含 traceId，支持全链路追踪（通过 Gateway 生成并透传）

## 8. 安全控制规范

### 8.1 认证流程

```
客户端 → Gateway(JWT验签) → 路由到服务 → 服务解析JWT获取用户信息
```

- JWT Token 有效期：24小时
- RefreshToken 有效期：7天
- Token 存储在 Redis 黑名单中实现登出

### 8.2 鉴权控制

- Gateway 层：统一验签 + 角色校验（admin/user）
- 服务层：AOP 切面校验具体操作权限
- Feign 调用：通过 RequestInterceptor 自动传递 JWT Token

### 8.3 安全约束

| 约束 | 说明 |
|------|------|
| 密码加密 | BCrypt，强度10 |
| SQL 注入防护 | MyBatis-Plus 参数化查询，禁止拼接SQL |
| XSS 防护 | 输入过滤 + 输出转义 |
| 敏感数据 | 身份证号、手机号在日志中脱敏 |
| 接口限流 | Gateway + Sentinel 双层限流 |
| HTTPS | 生产环境强制使用 |
