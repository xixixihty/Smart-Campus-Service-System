## 1. 系统概述

### 1.1 项目背景

智慧校园服务系统是一个综合性的校园管理平台，涵盖教务管理、图书管理、座位预约、通知公告等核心功能，为学校师生提供一站式的校园服务。

### 1.2 用户角色定义

| 角色类型 | 说明 | 访问端 |
|---------|------|--------|
| 管理员（Admin） | 系统管理员，拥有所有管理权限 | 管理端 |
| 教师（Teacher） | 授课教师，可进行审批、成绩录入等操作 | 管理端 + 用户端 |
| 学生（Student） | 在校学生，使用系统各项服务 | 用户端 |

### 1.3 技术架构

| 组件 | 技术选型 |
|------|---------|
| 后端框架 | Spring Boot 3 + MyBatis |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis（含 Redisson 分布式锁） |
| 消息队列 | RabbitMQ |
| API文档 | SpringDoc OpenAPI（Swagger UI） |
| 分页 | PageHelper |
| 认证 | JWT Token（jjwt 0.12.x） |
| 密码加密 | BCrypt |
| 对象存储 | 阿里云 OSS |

### 1.4 接口路径规范

本系统采用统一的路径前缀规范，所有接口均以 `/api/` 开头：

| 端分类 | 路径格式 | 示例 |
|--------|---------|------|
| 管理端接口 | `/api/{resource}/admin` | `/api/colleges/admin`、`/api/students/admin` |
| 用户端接口 | `/api/{resource}/user` | `/api/course-selections/user`、`/api/leave-requests/user` |
| 公共接口 | `/api/auth/*` | `/api/auth/login` |

> **注意**：用户端通知模块路径为 `/api/notices`（无 `/user` 后缀），属于历史遗留，后续版本建议统一。

## 2. 管理端功能模块

> 管理端功能面向管理员和教师，提供系统配置、数据管理和审批操作能力。所有管理端接口路径格式为 `/api/{resource}/admin`。

#### 2.1.2 涉及表

- `major`（专业表）
- `class`（班级表）

| 操作 | 方法 | 路径 | 请求参数 | 返回值 |
|------|------|------|----------|--------|
| 用户登录 | POST | `/api/auth/login` | `LoginDTO`(请求体) | `LoginResponseDTO` |

**LoginDTO 请求示例**：
```json
{
  "username": "T2021001",
  "password": "Password123!"
}
```

**LoginResponseDTO 响应示例**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "userId": 1,
    "userName": "张伟",
    "userType": "teacher"
  }
}
```

## 5. 接口规范

### 5.1 通用响应格式

所有接口统一使用 `Result<T>` 包装返回：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功 |
| message | String | 提示信息 |
| data | T | 响应数据，可为对象、数组或null |

### 5.2 分页响应格式

分页查询返回 `PageInfo<T>`：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 100,
    "list": [],
    "pageNum": 1,
    "pageSize": 10,
    "pages": 10
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| total | Long | 总记录数 |
| list | List\<T\> | 当前页数据 |
| pageNum | Integer | 当前页码 |
| pageSize | Integer | 每页条数 |
| pages | Integer | 总页数 |

### 5.3 状态码定义

| 状态码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 400 | 请求参数错误 |
| 401 | 未认证（Token无效或过期） |
| 403 | 无权限 |

- **时间格式（仅时间）**：`HH:mm`（如 `10:30`）

## 6. 业务规则

### 6.1 认证与授权

1. 学生使用学号登录，教师使用工号登录，统一使用 `/api/auth/login` 接口
2. 密码使用BCrypt加密存储，不可逆
3. JWT Token有效期默认24小时
4. 修改密码需验证旧密码
5. 管理员可重置学生/教师密码

### 6.2 组织架构规则

1. 学院 → 专业 → 班级，三级层级关系
2. 删除学院前需确保无关联专业
3. 删除专业前需确保无关联班级
4. 状态为"禁用"的组织节点不可使用

### 6.3 排课规则

1. 同一教室同一时间段不可重复排课
2. 同一教师同一时间段不可重复排课
3. 同一班级同一时间段不可重复排课
4. 排课冲突检测在创建和更新时自动触发
5. 星期几取值范围：1-7（周一至周日）
6. 节次范围：1-12

### 6.4 选课规则

1. 学生只能在选课时间段内进行选课
2. 同一学生同一课程不可重复选课
3. 退课后可重新选课（在选课时间段内）
4. 选课状态：已选 / 退课
5. 选课过程使用 Redisson 分布式锁 + Redis Lua 脚本保证并发安全
6. 选课结果通过 RabbitMQ 异步落库

### 6.5 成绩规则

1. 总评成绩 = 平时成绩 × 30% + 期末成绩 × 70%
2. 考试状态：正常 / 缓考 / 缺考 / 作弊 / 补考
3. 缺考或作弊的学生可安排补考
4. 补考成绩录入后更新原成绩记录
5. 绩点计算：90-100→4.0, 80-89→3.0, 70-79→2.0, 60-69→1.0, <60→0

### 6.6 请假规则

1. 请假类型：事假 / 病假 / 公假
2. 请假状态流转：待审批 → 已批准 / 已驳回
3. 只有"待审批"状态的请假可取消
4. 审批操作记录到 `leave_approval_log` 表
5. 审批结果：批准 / 驳回

### 6.7 通知规则

1. 发布范围类型：全部 / 学院 / 专业 / 班级
2. 通知状态：发布 / 撤回
3. 撤回后的通知不可再编辑
4. 用户端只显示与自己相关的通知（按发布范围过滤）

### 6.8 图书借阅规则

1. 图书状态：在库 / 借出 / 维修 / 遗失
2. 借阅时 `availableCopies` 减1
3. 归还时 `availableCopies` 加1
4. 借阅期限默认30天
5. 逾期未还将标记为"逾期"状态

### 6.9 座位预约规则

1. 座位状态：空闲 / 使用中 / 暂离 / 维修
2. 预约状态流转：待签到 → 使用中（签到后）→ 已完成（签退后）
3. 预约后需在规定时间内签到，否则标记为"违约"
4. 暂离有时间限制，超时自动签退
5. 同一座位同一时间段不可重复预约
