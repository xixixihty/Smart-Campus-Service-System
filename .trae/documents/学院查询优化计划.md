# 学院查询功能优化计划

## 1. 优化背景

用户要求优化学院管理模块的查询功能，采用软删除方案后，希望获取学院列表和查询详情的功能只查询状态为"启用"的学院。

## 2. 优化目标

1. **列表查询优化**：`getCollegeList`方法只返回状态为"启用"的学院
2. **详情查询优化**：`getCollegeDetail`方法只返回状态为"启用"的学院
3. **保持兼容性**：确保其他功能（如更新、删除、状态更新）不受影响

## 3. 优化内容

### 3.1 Service层优化
- 修改`CollegeServiceImpl.getCollegeList`方法，默认只查询状态为"启用"的学院
- 修改`CollegeServiceImpl.createCollege`方法，确保新创建的学院状态为"启用"

### 3.2 Mapper层优化
- 修改`CollegeMapper.getCollegeList`方法，添加状态条件
- 修改`CollegeMapper.getCollegeDetail`方法，添加状态条件

### 3.3 常量定义
- 确保使用统一的状态常量：`COLLEGE_STATUS_NORMAL`表示启用状态

## 4. 优化步骤

1. **分析当前实现**：查看`CollegeMapper.xml`中的SQL实现
2. **修改Mapper接口**：更新查询方法的参数和实现
3. **修改Service实现**：更新业务逻辑
4. **测试验证**：确保查询功能只返回启用状态的学院

## 5. 预期成果

- 学院列表查询只返回状态为"启用"的学院
- 学院详情查询只返回状态为"启用"的学院
- 其他功能（如更新、删除、状态更新）正常运行
- 代码符合阿里巴巴开发手册规范
